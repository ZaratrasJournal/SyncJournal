Trades samenvoegen FTMO MT5 partial-workaround

FTMO web staat geen partial-TPs toe — je opent meerdere losse trades als workaround, maar het journal ziet er dan 3-4 aparte trades i.p.v. 1 positie met 3-4 TPs. Deze feature voegt geselecteerde trades samen tot één master-trade met mergedFrom: [id1,id2,id3,id4]. De originelen blijven bewaard (soft-merge), worden uit lijsten + analytics gefilterd, en de master krijgt een 🔗-badge.

🔧 Tech-notes — wat slaat de app op?

Master-trade velden

  • entry = Σ(child.entry · size) / Σ(size) — gewogen avg
  • size = Σ(child.size) — totale notional
  • exit = gewogen avg van closed-children exits
  • pnl = Σ(child.pnl) — netto winst alle children
  • fees = Σ(child.fees)
  • sl = conservatiefste SL (min afstand per direction)
  • date = vroegste child.date
  • status = closed als alle children closed; partial anders
  • mergedFrom: [id, id, ...]
  • _mergeSource: "manual"

Children-velden bij merge

  • status → "merged-child"
  • mergedInto: masterId
  • _preMergeStatus bewaard voor exact restore

Filter-discipline

  • Nieuwe helper visibleTrades(trades) excludeert status==="merged-child"
  • Overal toepassen: Trades-tabel, Dashboard, Analytics, Calendar — anders dubbeltel-bug zodra eerste merge bestaat

Validatie (block / warn)

  • Block: verschillende pair / direction / source / ≥2 master-trades / merged-child al gemerged
  • Warn (overrulebaar): tijdspreiding >24u, deels open trades
  • Vraag: conflict op tradeGrade / playbookId / notes

De-merge

  • Master verwijderen (geen zombie-state)
  • Children krijgen mergedInto:null + status restored via _preMergeStatus
  • Knop in master-detail: "🔓 Splits weer uit"

Scope v1

  • Knop verschijnt alleen bij source==="ftmo"
  • Banner in modal: "voor crypto-exchanges herkent de app partials automatisch"

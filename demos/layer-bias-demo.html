Setup-laag bias — 5-fasen feature-plan

Visualiseert wat we gaan bouwen over v12.203 → v12.207. Per laag krijg je een bullish/neutraal/bearish-bias. De app classifeert dan automatisch Continuation / Pullback / Reversal / Counter op basis van hoe de bias-flow zich verhoudt tot de trade-direction. Analytics + playbook-stats laten dan zien welke patronen voor jou werken en welke je moet vermijden.

Fase 2 v12.204  Pattern-classificatie + badge in Trades-tabel

Helper getTradeAlignmentPattern(trade) classifeert elke trade als Continuation / Pullback / Reversal / Counter / Unclassified op basis van hoe HTF + LTF lagen zich verhouden tot trade-direction. Badge naast de Setup-kolom toont direct welk patroon. Trades-filter: "Toon alleen [pattern]".

Classifier-logica (Optie II — TF-rol gevoelig):
• Continuation: alle HTF + alle LTF bias matched met trade-direction.
• Pullback: HTF aligned met direction, LTF tegen direction (= klassieke retracement-entry).
• Reversal: HTF tegen direction, LTF aligned (= turn-call op LTF).
• Counter: zowel HTF als LTF tegen direction.
• Unclassified: geen HTF-lagen OF geen bias-velden ingevuld.
Filter "Toon alleen [pattern]" in de FilterBar — direct edge-analyse mogelijk.

Recap — wat de 5 fases samen opleveren

  • v12.203: bias-veld + UI (cosmetic, geen analytics)
  • v12.204: classifier + badge + filter (zonder data nog visueel onderscheid)
  • v12.205: KPI-strip + WR-chart (eerste echte edge-inzicht)
  • v12.206: HTF×LTF heatmap (granulaire combinatie-analyse)
  • v12.207: playbook bias-signature + distributie (per setup zien waar de lekken zitten)

Elke release op zichzelf shippable. Fase 1 = laag risico (puur schema). Fases 3-4 = analytics-uitbreidingen (middel risico, theme-tests vereist). Pre-trade compliance (real-time warning) blijft expliciet buiten dit plan — eerst data verzamelen, dan slim worden.

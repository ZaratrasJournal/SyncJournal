# SyncJournal UX-audit — v12.154

**Datum**: 2026-06-01
**Scope**: 7 primaire schermen (Dashboard, Trades, Analytics, Playbook, AI-coach, Instellingen, Trade-form, Chat-popup) × 2 thema's (sync / light)
**Methodologie**: heuristische evaluatie (Nielsen 10) + visual audit (Refactoring UI) + information architecture + code-metrics
**Auditor**: Claude (senior UX/UI-designer)
**Evidence**: 16 screenshots in `tests/screenshots/ux-audit/` + code-grep metrics in dit document

---

## 1. Executive summary

De app heeft een **sterke fundering** (goed kleur-token-systeem, leesbare tabellen, mooie Playbook-cards) maar lijdt aan **drie systemische issues** die elk individueel scherm minder samenhangend maken dan ze zouden kunnen zijn.

### Issue 1 — Typografie zonder schaal
Er zijn **23 verschillende font-sizes** in gebruik (`8/9/9.5/10/10.5/11/11.5/12/12.5/13/14/15/16/18/20/22/24/26/28/30/32/36/48px`). Top 4: `11px (309×)`, `10px (254×)`, `12px (216×)`, `9px (123×)`. Elke +0.5px stap is iemand die improviseert op "iets kleiner / iets groter". Het kleur-token-systeem (`--text/text2/text3/text4/text5`) is exemplary, maar er is **geen tegenhanger voor type**. Gevolg: in de KPI-blokken van Dashboard en Analytics staan 100% / 100% / 96% met cijfers van ~48px naast sub-stats van 9-10px — twee orden van grootte verschil zonder midden-niveau dat hiërarchie genereert.

### Issue 2 — Filter-bar overload op Trades + Analytics
Beide schermen openen met **7 rijen filter-controls** vóór de eigenlijke data zichtbaar wordt: (1) date-from/to, (2) preset-pills (Vandaag/Week/Maand), (3) Long/Short/Winners/Losers, (4) pair-pills (5+), (5) account-pills (5), (6) "Geavanceerde filters" disclosure, (7) zoek-input + results-summary. Dit eet ~280-320px lodrechte ruimte op een 900px viewport — **35% van de visible-area wordt filter, niet data**. Plus de zoek-input — een primary action — staat op rij 4, verstopt onder pills.

### Issue 3 — Topbar-rechts indicator-creep
In `tj-right-btns` stapelen zich nu op: Trade-knop + Backup-indicator + Weekly-due + Budget-warning + Privacy-toggle + Thema-toggle + Mori-FAB = **6-7 elementen**. Op een 1440px-viewport past dat, op een 1100px laptop wordt het al krap, op mobile (<600px) is alles weg behalve de Trade-knop. Er is geen voorkeurs-strategie: welke indicator mag opzij wanneer? Welke is hoofdactie, welke is alert?

**Aanbevolen aanpak**: introduceer een type-scale + spacing-scale-token-systeem (~2u), refactor de filter-bar naar 1 hoofdrij + collapsed advanced (~3u), consolideer indicators naar "primary + alert + menu" (~1u). Samen ~6u werk dat de gevoel-van-rommel op 80% van de schermen wegneemt zonder feature-functionaliteit te raken.

## 3. Design system tweaks (de "shared" fixes)

### 3.1 Type-scale token-systeem (NIEUW — grootste impact)

Introduceer 6 type-tokens in de bestaande `<style>` block, gebruik ze via `font-size:var(--text-xs)` etc. Vervang de huidige 23 varianten:

```css
:root {
  --text-xs:   11px;  /* labels, badges, micro-info — vervangt 9/9.5/10/10.5 */
  --text-sm:   12px;  /* body-meta, table-cells, secondary copy */
  --text-base: 13px;  /* primary body — vervangt 12/12.5/13 */
  --text-lg:   15px;  /* section headers, important labels */
  --text-xl:   18px;  /* page headers, modal titles */
  --text-2xl:  24px;  /* hero numbers (KPI hero, ticker) — vervangt 22/24/26/28 */
  --text-3xl:  32px;  /* dashboard-hero (huidige $67k ticker) */
}
```

**Migratie-plan**: alle `<text-xs (9-10.5px)` → `--text-xs`. Alle 11-13px → `--text-sm` of `--text-base`. Alles >22px → `--text-2xl` of `--text-3xl`. 9.5/10.5/11.5/12.5px (in totaal ~50 occurrences) krijgen `--text-xs` of `--text-sm` afhankelijk van context. **Effect**: van 23 → 6 sizes. Visuele consistentie als bijproduct.

### 3.2 Spacing-scale token-systeem (NIEUW)

4pt grid, 5 niveaus. Vervangt de 10 huidige gap-waarden en 13 padding-waarden:

```css
:root {
  --space-1: 4px;   /* tight gaps in flex-rows */
  --space-2: 8px;   /* default button-gap, list-item internal */
  --space-3: 12px;  /* card-padding internal */
  --space-4: 16px;  /* section-internal padding */
  --space-5: 24px;  /* between-section spacing */
  --space-6: 32px;  /* major page-section breaks */
}
```

### 3.3 Button-tiers (consolideren naar 3)

Vervang de 15+ button-bg-varianten met 3 strikte tiers:

| Tier | Wanneer | Stijl | Voorbeeld |
|------|---------|-------|-----------|
| **Primary** | hoofd-actie per scherm, max 1 zichtbaar | `bg:var(--gold)` + `color:var(--bg)` + solid border | "Trade opslaan", "Download backup nu" |
| **Secondary** | sub-acties, kunnen er meerdere zijn | `bg:var(--gold-dim)` + `color:var(--gold)` + gold-border | "Test verbinding", "Kies folder" |
| **Tertiary** | dismiss, cancel, less-important | `bg:transparent` + `color:var(--text3)` + `border:var(--border3)` | "Annuleren", "Sluiten" |

Schrap de tussenliggende `rgba(255,255,255,.02-.10)` backgrounds — die zijn alleen visuele ruis.

### 3.4 borderRadius — reduceren van 13 naar 4

```css
:root {
  --radius-sm:    6px;   /* buttons, inputs, badges */
  --radius-md:    10px;  /* cards, modals (huidige --radius is 12, naar 10 brengen) */
  --radius-lg:    14px;  /* major cards (playbook, popup) */
  --radius-pill:  999px; /* status-badges, pills */
}
```

Schrap: 2/3/4/5/9/16/18px varianten.

### 3.5 Indicator-strategie voor topbar

**Regel**: max **3** items rechts in topbar:
1. **Trade-knop** (altijd, primary action)
2. **Alert-bubble** (max 1 actief — prioriteit: budget-blocked > backup-overdue > weekly-due)
3. **Settings-pill** (☰ menu met overige: privacy, thema, Mori-FAB)

Het Mori-FAB blijft rechtsonder als floating-button — niet in topbar. Backup-indicator + Weekly + Budget consolideren in **één** "🔔 N" bubble die bij klik een dropdown opent met alle relevante alerts.

### 3.6 Empty-state component (consistentie)

Standaardstructuur voor de 9 verschillende "geen X"-states (huidige varianten zijn allemaal ad-hoc):

```jsx
<EmptyState
  icon="📋"
  title="Geen trades gevonden"
  body="Pas je filters aan of voeg een trade toe."
  action={<button>+ Nieuwe trade</button>}
/>
```

Eén component, één layout, één type-spacing. Voorbeelden in code: regel 3765 ("Nog geen trades"), 3998 ("Geen trades gevonden"), 5014 ("Geen trades in deze periode") — allen ander wording en stijl voor in essentie dezelfde situatie.

## 5. Voor / na — top 3 visuele fixes

### Fix 1 — Dashboard KPI-hiërarchie

**Voor** (huidig):
```
┌──────────────────────────────────────────────┐
│  MAAND-DOELEN                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ MAAND PNL│  │ WIN RATE │  │  MAX DD  │    │  ← 3 tiles, allen gelijk
│  │   +$50   │  │   0%     │  │   $50    │    │  ← cijfer ~32px
│  │ Doel:    │  │ Doel:    │  │ Doel:    │    │  ← sub ~9px (UNREADABLE)
│  │ +$2,000  │  │ 60%      │  │ $500     │    │
│  └──────────┘  └──────────┘  └──────────┘    │
└──────────────────────────────────────────────┘
```

**Na**:
```
┌──────────────────────────────────────────────┐
│  MAAND-DOELEN                                │
│  ┌─────────────────────┐  ┌──────────┐       │
│  │  MAAND PNL          │  │ WIN RATE │       │  ← Hero 2/3 width
│  │                     │  │   0%     │       │
│  │       +$50          │  │ ─────    │       │  ← cijfer 24-32px
│  │   ───────           │  │ MAX DD   │       │
│  │   Doel +$2.000      │  │   $50    │       │  ← sub 12-13px (LEESBAAR)
│  │   Voortgang: 2.5%   │  │          │       │
│  └─────────────────────┘  └──────────┘       │
└──────────────────────────────────────────────┘
```

Win: oog gaat eerst naar het belangrijkste cijfer (Maand PnL), secundaire metrics blijven zichtbaar maar nemen minder visueel gewicht. Geen onleesbare 9px-tekst.

### Fix 2 — Trades filter-bar collapse

**Voor**:
```
┌────────────────────────────────────────────────────────────────────┐
│ [dd-mm-jj][dd-mm-jj] [Vandaag][Week][Maand] [Long][Short][Win][Loss]│  ← rij 1
│ [AVA/USDT][BTC/USDT][BTC/USD][DOGE][ETH/USDT]                       │  ← rij 2
│ [Blofin-Swing] [FTMO] [Hyperliquid] [Kraken] [MEXC]                 │  ← rij 3
│ ▾ Geavanceerde filters                                              │  ← rij 4
│ [🔍 Zoek op pair, datum, notes... (bijv: BTC, FOMO, 2R)]            │  ← rij 5
│ 788 trades · +$29,912.49 · WR 50.6%                                 │  ← rij 6
├────────────────────────────────────────────────────────────────────┤
│ (data begint hier — na ~280px scroll)                               │
```

**Na**:
```
┌────────────────────────────────────────────────────────────────────┐
│ [dd-mm-jj] - [dd-mm-jj]  [🔍 Zoek pair/notes/tags...] [▾ 3 filters] │  ← 1 rij
│ 788 trades · +$29,912.49 · WR 50.6%                                 │  ← results
├────────────────────────────────────────────────────────────────────┤
│ (data begint hier — direct zichtbaar)                               │
```

Win: 280px → 80px verloren voor filters. "▾ 3 filters" bubble toont aantal-actief; klik = popout met alle preset-pills/pair/account-pills. Search is primary action. Geen verlies aan functionaliteit, wel aan visuele ruis.

### Fix 3 — Topbar indicator-consolidation

**Voor** (right-side van topbar, 6 elementen):
```
... [Playbook] [AI-coach] [Instellingen]   [+ Trade N] [📦 8d] [📅 due] [💰 80%] [👁] [☀] [🤖]

                                                                                          Mori FAB
```

**Na** (right-side, 3 elementen + Mori FAB blijft rechtsonder):
```
... [Playbook] [AI-coach] [Instellingen]   [+ Trade N] [🔔 3] [⋯]
                                                       ↑      ↑
                                                  alle alerts  privacy + thema
                                                  in dropdown    in menu
```

Win: topbar gaat van 6 → 3 elementen rechts. Schaalbaar voor toekomstige indicators. `🔔 3` bubble bij klik:
```
┌─────────────────────────────────┐
│ 💰 Budget 80% gebruikt          │
│ 📅 Weekly digest beschikbaar    │
│ 📦 Backup 8 dagen oud           │

```

## 6. Wat NIET veranderen — bewust behouden

- **Kleur-token-systeem** (`--text/text2-5`, `--bg/bg2-4`, `--border1-6`, `--gold/red/green/amber`) — exemplary, alle thema's werken
- **6 thema's** — laat dit groeisysteem heel; nieuwe tokens komen erbij, oude niet weg
- **Playbook-pagina** — beste design in de app, niet aanraken
- **Status-badges in Trades** (WIN/LOSS/BT/PAPER met kleur-coding) — duidelijk en consistent
- **Markdown rendering in Mori chat** — top
- **Single-file HTML constraint** — geen framework, geen build, geen utility-css-library
- **Power-user dichtheid in tabellen** — niet "luchtiger maken om mooi te zijn"; traders willen veel data tegelijk zien
